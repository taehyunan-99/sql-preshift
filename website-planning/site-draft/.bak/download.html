Install

Three steps to install.

From the downloaded disk image to a launched app. The first launch needs one extra confirmation because the build is unsigned — covered right after.

Open the disk image

Double-click the downloaded SQLPreShift-*.dmg. A Finder window opens showing the app and an Applications shortcut.

Drag the app to Applications

Drag SQLPreShift.app onto the Applications folder in the same window. This copies it into /Applications.

Open it for the first time

Because the build is unsigned, a plain double-click is blocked the first time. Use one of the two methods in the next section — you only need to do this once.

First launch

Get past Gatekeeper, once.

SQLPreShift is not notarized, so macOS blocks the first launch. This is expected. Pick the method that matches your macOS version — after the first open, it launches normally with a double-click.

Unsigned build

This is a portfolio app and is not Apple-notarized, so the first launch is blocked by Gatekeeper. That is normal. You only need to clear it once; every launch after opens with a double-click.

Right-click the app

In /Applications, right-click (or Control-click) SQLPreShift.app and choose Open.

Confirm in the dialog

A warning appears with an Open button. Click Open again. The app launches and is remembered as an exception.

Why this differs

On macOS Sequoia (15) the right-click · Open shortcut no longer overrides Gatekeeper. Use the Privacy & Security panel instead.

Try to open the app once

Double-click SQLPreShift.app. macOS shows a blocked dialog. Dismiss it.

Open System Settings · Privacy & Security

Scroll to the Security section at the bottom. You'll see a line: "SQLPreShift" was blocked…

Click Open Anyway

Then confirm Open in the follow-up dialog and enter your login password. Note the button is only offered for about an hour after the blocked attempt.

Remove the quarantine attribute directly. This is the fastest route if you're comfortable in a terminal.

Terminal
xattr -dr com.apple.quarantine /Applications/SQLPreShift.app
Security tip

This clears the quarantine flag for this app only (-r recursive, -d delete). Avoid spctl --global-disable, which weakens Gatekeeper for every app on the system.

Optional

Enable natural language to SQL

The safety core runs without any LLM. If you want plain-English input, install Ollama and pull two local models. Inference stays entirely on your machine.

Terminal — NL → SQL
ollama pull gemma4:latest   # natural language to SQL
Terminal — RAG embeddings
ollama pull bge-m3:latest   # RAG embeddings (1024-dim)
No Ollama? No problem

Without Ollama, SQLPreShift prompts you to type SQL directly. Every safety feature — dry-run stack, 18 risk rules, diagnostics — works with no LLM at all.

Run from source

Prefer to build it yourself?

Clone the repo and bring the whole stack up with Docker Compose. Demo databases are included so you can try the full flow without connecting your own data.

Terminal — clone & run
# from the cloned repo root
cp .env.example .env
docker compose up -d   # postgres + backend + frontend

Before you ask

Download FAQ

Why is the first launch blocked?

The build is unsigned and not notarized by Apple, so Gatekeeper blocks the first open. This is expected for a portfolio app. Clear it once with the right-click · Open method, the Privacy & Security panel on Sequoia, or the xattr command — afterward it opens normally.

Does it work on Intel Macs?

No. SQLPreShift targets Apple Silicon (M1 and newer) only. Check your chip under the Apple menu · About This Mac.

Do I have to install Ollama?

No. Ollama is only needed for natural-language input. Without it, you enter SQL directly and every safety feature still works.

Which databases does it support?

PostgreSQL 16 only. The included demo stack ships a 92-table ERP database and Pagila so you can try it without your own data.

Are my credentials stored anywhere?

No. The app is stateless and one-shot — connection credentials are not persisted between sessions, and inference (if you use the local LLM) stays on your machine.
