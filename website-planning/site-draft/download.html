설치

설치는 세 단계.

다운로드한 디스크 이미지에서 실행되는 앱까지. 빌드가 미서명 상태이기 때문에 첫 실행에는 추가 확인이 한 번 필요합니다. 바로 다음 섹션에서 다룹니다.

디스크 이미지 열기

다운로드한 SQLPreShift-*.dmg를 더블 클릭하세요. 앱과 응용 프로그램(Applications) 단축 아이콘이 표시된 Finder 창이 열립니다.

앱을 응용 프로그램으로 드래그

같은 창에서 SQLPreShift.app을 응용 프로그램(Applications) 폴더 위로 드래그하세요. /Applications로 복사됩니다.

처음 한 번 실행하기

빌드가 미서명 상태라 처음에는 단순 더블 클릭이 차단됩니다. 다음 섹션의 두 가지 방법 중 하나를 사용하세요. 이 작업은 한 번만 하면 됩니다.

첫 실행

Gatekeeper, 한 번만 통과하기.

SQLPreShift는 공증(notarize)되지 않았기 때문에 macOS가 첫 실행을 차단합니다. 정상적인 동작입니다. macOS 버전에 맞는 방법을 선택하세요. 한 번 연 뒤부터는 더블 클릭으로 평소처럼 실행됩니다.

미서명 빌드

이 앱은 포트폴리오 앱이며 Apple 공증을 받지 않았기 때문에 첫 실행이 Gatekeeper에 의해 차단됩니다. 정상입니다. 한 번만 해제하면 되며, 이후 모든 실행은 더블 클릭으로 열립니다.

앱을 우클릭

/Applications에서 SQLPreShift.app을 우클릭(또는 Control-클릭)한 뒤 열기(Open)를 선택하세요.

대화상자에서 확인

열기(Open) 버튼이 있는 경고 창이 나타납니다. 열기(Open)를 다시 클릭하세요. 앱이 실행되고 예외로 기억됩니다.

왜 다른가

macOS Sequoia(15)에서는 우클릭 · 열기(Open) 단축 동작이 더 이상 Gatekeeper를 우회하지 못합니다. 대신 개인정보 보호 및 보안(Privacy & Security) 패널을 사용하세요.

앱을 한 번 열어보기

SQLPreShift.app을 더블 클릭하세요. macOS가 차단 대화상자를 표시합니다. 이를 닫으세요.

시스템 설정(System Settings) · 개인정보 보호 및 보안(Privacy & Security) 열기

맨 아래 보안(Security) 섹션까지 스크롤하세요. "SQLPreShift" was blocked…라는 문구가 보입니다.

그래도 열기(Open Anyway) 클릭

이어지는 대화상자에서 열기(Open)를 확인하고 로그인 암호를 입력하세요. 이 버튼은 차단 시도 후 약 한 시간 동안만 제공됩니다.

격리(quarantine) 속성을 직접 제거합니다. 터미널 사용이 익숙하다면 가장 빠른 방법입니다.

터미널
xattr -dr com.apple.quarantine /Applications/SQLPreShift.app
보안 팁

이 명령은 해당 앱에 한해서만 격리 플래그를 제거합니다(-r 재귀, -d 삭제). 시스템의 모든 앱에 대해 Gatekeeper를 약화시키는 spctl --global-disable은 사용하지 마세요.

선택 사항

자연어를 SQL로 변환 활성화

안전 코어는 LLM 없이도 동작합니다. 자연어 입력을 사용하려면 Ollama를 설치하고 로컬 모델 두 개를 받으세요. 추론은 전적으로 사용자 머신에서 이루어집니다.

터미널 · NL to SQL
ollama pull gemma4:latest   # 자연어를 SQL로
터미널 · RAG 임베딩
ollama pull bge-m3:latest   # RAG 임베딩 (1024-dim)
Ollama가 없어도 괜찮습니다

Ollama 없이도 SQLPreShift는 SQL을 직접 입력하도록 안내합니다. dry-run 스택, 17개 위험 규칙, 진단 등 모든 안전 기능이 LLM 없이도 동작합니다.

소스에서 실행

직접 빌드하고 싶으신가요?

저장소를 클론하고 Docker Compose로 전체 스택을 기동하세요. 데모 데이터베이스가 포함되어 있어 자신의 데이터를 연결하지 않고도 전체 흐름을 체험할 수 있습니다.

터미널 · 클론 & 실행
# 클론한 저장소 루트에서
cp .env.example .env
docker compose up -d   # postgres + backend + frontend

데모 데이터베이스

이 스택에는 시드 데이터가 적재된 PostgreSQL 데이터베이스 두 개가 함께 제공되어, 바로 게이트해볼 수 있는 현실적인 데이터를 제공합니다.

  • ERP 데모 · 92개 테이블 · localhost:5433
  • Pagila · 영화 대여 샘플 스키마 · localhost:5434

로컬에 머무는 것

이 앱은 상태를 저장하지 않고 일회성으로 동작하며, 자격증명은 세션 간에 절대 저장되지 않습니다.

  • 단일 SQLite 메타데이터 저장소
  • PostgreSQL 전용 (16에서 검증)
  • 포트폴리오 데모 · 호스팅 서비스 아님

다운로드 FAQ

첫 실행이 왜 차단되나요?

빌드가 미서명 상태이며 Apple 공증을 받지 않았기 때문에 Gatekeeper가 첫 실행을 차단합니다. 포트폴리오 앱에서는 정상적인 동작입니다. 우클릭 · 열기(Open) 방법, Sequoia의 개인정보 보호 및 보안(Privacy & Security) 패널, 또는 xattr 명령으로 한 번만 해제하면, 이후에는 평소처럼 열립니다.

Intel Mac에서도 동작하나요?

아니요. SQLPreShift는 Apple Silicon(M1 이상) 전용입니다. Apple 메뉴 · 이 Mac에 관하여(About This Mac)에서 칩을 확인하세요.

Ollama를 꼭 설치해야 하나요?

아니요. Ollama는 자연어 입력에만 필요합니다. 없어도 SQL을 직접 입력하면 되며 모든 안전 기능이 그대로 동작합니다.

어떤 데이터베이스를 지원하나요?

PostgreSQL 전용이며, PostgreSQL 16에서 빌드·검증했습니다. 포함된 데모 스택에는 92개 테이블의 ERP 데이터베이스와 Pagila가 함께 제공되어, 자신의 데이터 없이도 사용해볼 수 있습니다.

제 자격증명이 어딘가에 저장되나요?

아니요. 이 앱은 상태를 저장하지 않고 일회성으로 동작하며, 연결 자격증명은 세션 간에 저장되지 않고, 추론(로컬 LLM을 사용하는 경우)도 사용자 머신에 머뭅니다.

확인을 마쳤다면, 이제 게이트를 열 차례입니다.

무료 오픈소스. 모든 작업이 로컬에서 동작합니다. 운영 DB에 ALTER를 던지는 순간을 위해 만들었습니다.
