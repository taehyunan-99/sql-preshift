# t4-tab-hub-bone

IA 유형 D(탭/허브형)를 충실히 구현: 최상위 nav가 콘텐츠 종류별 탭(Hub / Product / Download / Open Source)이고, 허브 랜딩(index)은 4개 영역을 번호 매긴 카드 그리드로 진입시키는 디스패처(설득보다 길잡이)다. 각 탭 페이지(product/download/open-source)는 자기만의 보조 사이드바를 가져 탭 전환 시 레이아웃이 통째로 바뀌는 'N개 미니 사이트의 허브' 구조 — 단일 트리나 동일 nav 독립 페이지와 구별된다. 색 테마는 웜 본(#F2EDE6 캔버스 / #FBF8F4 카드 surface) + 강한 coral(#E2603E)의 하이콘트라스트 라이트로, charcoal #232020 텍스트에 warm-cast shadow(rgba(74,48,30,...))로 카드 elevation 위계를 줘 graphite·teal 고정을 완전히 벗어났다. 이식한 레퍼런스 패턴: (1) Proxyman 플랫폼 카드 그리드를 라이트 변형으로 → 허브 디스패처 카드 + Download 페이지 hero급 단일 다운로드 카드. (2) Apple 102445 + TrozWare/iDownloadBlog 교차확인을 그대로 반영해 Gatekeeper를 OS 버전 분기 탭으로(Sequoia=Privacy & Security 'Open Anyway' 4단계 / Sonoma 이하=우클릭 Open / Terminal=xattr -dr) 구성하고 'right-click no longer works on Sequoia'를 명시. (3) Raycast Manual + Beekeeper docs의 좌측 트리 사이드바 + Microsoft style '기대 결과 한 줄'을 작동 흐름 7단계 Steps에 적용. (4) Bruno/Proxyman의 Info/Warning/Tip + Starlight danger Asides 4종 콜아웃으로 안전성 주장 분리('Apply All은 단일 TX', 'This changes nothing', 미공증 honest disclosure). (5) Beekeeper의 OSS 프레이밍을 Open Source 탭으로 — Pricing 슬롯을 'Free by design'으로 대체. 스크린샷 자리는 전부 bone+coral 톤 CSS 목업(ERD Split diff, 리스크 모달+golden path, dry-run 스택, size-aware 바, 진단 리스트, NL→SQL)으로 채웠고, 코드블록은 monospace + 복사 버튼(JS clipboard) 동작.
