Why it exists

Not a daily tool. A gate you hit before the dangerous ones.

A migration that locks a 100-million-row table at peak traffic is not a bug you fix afterward — it is an outage. SQLPreShift is the checkpoint that reads the change against your real database first, so the regret never happens.

THE PROBLEM

Apply, then regret

Schema migrations look harmless in a text editor. The lock queue, the full table rewrite, the unvalidated constraint — none of it shows until it is already running in production.

THE GATE

Stop it before

Every change is dry-run against the live schema, weighed against 18 risk rules with estimated row counts, and stacked for review. Nothing commits until Apply All.

THE SCOPE

Honest about limits

PostgreSQL 16 only. Stateless and one-shot. A portfolio-grade demo, not a managed service — and it says so. Natural language is optional; the core works without an LLM.

Core features

Five things SQLPreShift does

The whole product is built so you read the consequences of a migration before you commit it.

01

Cumulative dry-run stack + single-transaction apply

Each change is dry-run against the live DB, pushed onto a stack, and reviewed with Undo. Apply All wraps the stack in one transaction — no half-applied migrations.

02

18 risk rules + golden-path alternatives

Detects lock-queue operations: DELETE/UPDATE without WHERE, DROP, full rewrites, constraint validation. It doesn't just block — it offers a zero-downtime path (ADD CONSTRAINT … NOT VALID then VALIDATE).

03

Size-aware impact

Every risk warning carries the target table's estimated row count and size. 100 rows and 100 million rows read very differently.

04

Read-only integrity diagnostics on connect

The moment you connect, four read-only diagnostics run — including broken referential integrity. Nothing is changed.

05 OPTIONAL

Local LLM for NL to SQL + RAG

A local Ollama turns natural language into SQL. Credentials and inference stay local. No Ollama? You're guided to enter SQL directly — the core works without the LLM.
