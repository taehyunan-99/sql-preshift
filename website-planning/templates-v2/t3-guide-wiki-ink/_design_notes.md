# t3-guide-wiki-ink

IA: 가이드-위키형(유형 C). 얕은 2단계 평면 IA 4그룹(01 Get started / 02 Core workflow / 03 Risk & safety / 04 Local LLM)을 좌측 사이드바에 고정해 lost-in-the-middle 차단 — docs 트리(A)나 탭 분기(D)가 아닌 '평면 목록'으로 구조를 차별화. index는 약한 hero + Quickstart 카드(Connect→첫 dry-run 4스텝) + Popular 6항목 고정으로 위키 진입점 역할. 나머지 3페이지는 breadcrumb + 위키링크식 인라인 상호참조([[Risk Rules]] 스타일, ::before/::after로 대괄호 렌더) + 우측 미니 TOC(scroll-spy) + 하단 prev/next의 밀도 높은 매뉴얼.
색 테마: 하이콘트라스트 잉크 — 배경 near-black #0E0E10(웜/보라기 없는 순잉크), 텍스트 #FAFAFA, ink-900~500 회색 위계만으로 구조. accent는 violet #7C5CFF 단 1색(위키링크·콜아웃 헤더·Quickstart·코드 키워드·스텝 번호 원형에만 강채도). graphite+teal 고정에서 색·구조 양쪽으로 확실히 이탈.
이식한 레퍼런스 패턴: (1) Obsidian /help의 4그룹×2단계 평면 위키 + 위키링크 인라인 상호참조. (2) Starlight/Tauri의 Steps 컴포넌트(번호 원형+수직 연결선)로 7단계 작동흐름, synced Tabs로 NL/SQL·dmg/source 분기, 4종 Asides(note/tip/caution/danger)로 안전성 주장 격리. (3) Apple 도움말 + Beekeeper의 짧은 번호 절차 + Gatekeeper OS 분기(우클릭=Sonoma 이하, Privacy&Security=Sequoia 이상, xattr=advanced). (4) Zed utilitarian 단색 회색위계 + Raycast식 ⌘K 검색 박스 + Warp docs의 좌사이드바·우TOC·prev/next 3구역 셸. (5) 소셜프루프 없는 자리를 정직한 trust 카피(single TX·100% local·read-only diagnostics)로 대체. ERD diff·Risk 모달·Dry-run 스택은 CSS 목업으로 잉크 톤에 맞춰 정성껏 구현(회색 박스 금지). 이모지·화살표 글리프 0, UI 카피 전부 영어.
